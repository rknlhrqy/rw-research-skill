# RW Research Question 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 比较干预效果 | 使用人群、干预、比较、结果和时间组织问题 | 省略比较条件 |
| 理解经验或过程 | 定义人群、现象和情境 | 强行加入干预 |
| 预测结局 | 定义预测时点、目标结局和验证层级 | 把预测写成因果 |
| 已有数据库 | 先做可回答性和偏倚检查 | 按字段拼问题 |
| 声称研究空白 | 交给文献发现核验 | 凭印象写 gap |

## 核心规则

- 先选择问题类型，再选择问题框架；PICO 不是所有问题的默认格式。
- 可检索的问题要把核心构念拆成概念和同义词。
- 可执行的问题要说明能够取得什么数据，而不是只写理想数据。
- 可证伪的问题要说明何种观察会削弱主要解释。
- 构念定义不清时，样本量和统计方法都不能补救。
- 数据已经存在，只能限制问题，不能自动证明问题有价值。
- 描述问题不能在输出中变成因果问题。
- 预测问题要区分模型开发、内部验证、外部验证和临床效用。
- 定性问题要说明经验、过程或意义所处的情境。
- 实施问题要区分干预效果和实施结果。
- 研究问题必须有范围边界，不能靠无限加变量解决。
- 最终问题应能生成检索式、纳排标准或研究设计入口。
