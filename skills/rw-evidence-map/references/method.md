# RW Evidence Map 方法

## 输入门

- 保留用户原问题和原始材料。
- 确认交付物、研究类型、可用证据和不能越过的范围。
- 缺少会改变判断的信息时，只追问必要项；其余内容继续处理。

## 执行阶段

1. 定义证据图服务的研究问题、决策和时间范围。
2. 为每项研究建立稳定 ID，并连接原文或可核验标识符。
3. 提取研究设计、人群、暴露或干预、比较、结果、时间和效应信息。
4. 单独记录偏倚风险、适用性、精确性和直接性。
5. 把关系标为支持、反驳、限制、不可比较或待核对。
6. 输出证据簇、冲突原因、空白和下一步取证。

## 证据门

- 每项事实能回到用户材料或可核验公开来源。
- 推断标记为推断，缺失标记为缺失，不用常识补成已知事实。
- 当前信息在使用当日核验；稳定方法使用 Skill 内的摘要。
- 报告规范、偏倚工具和研究设计各自承担不同功能。

## 输出契约

- 证据矩阵或关系图。
- 关系来源、偏倚和可比性记录。
- 冲突解释、缺口和下一步。

输出同时标记：已确认事实、当前判断、来源、未知事项、停止条件和下一步。

## 停止条件

- 摘要不足以判断方法或结果时，标记待全文核对。
- 缺少共同结果口径时，不做数值合并。
- 不把图中连接数量当成证据强度。
- 没有来源定位的关系只能进入候选区。
