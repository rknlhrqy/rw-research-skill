# RW PhD Write 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 只有主题没有来源 | 先建立论证和检索缺口 | 生成带引用正文 |
| 已有来源卡片 | 先做 claim-to-source 对齐 | 按文献逐篇总结 |
| 重写 Results | 保留数值、分析集和不确定性 | 加入解释 |
| 重写 Discussion | 区分结果解释、外部证据和限制 | 把相关性升为因果 |
| 需要个人语气 | 调用 rw-phd-tone 并提供样本 | 依赖本地隐藏语料 |

## 核心规则

- 写作从章节功能和主张开始，不从漂亮句子开始。
- 每个事实性主张都要有用户材料或可核验来源。
- 来源支持范围小于句子范围时，收窄句子。
- 结果、解释、推断和建议要使用不同的证据强度。
- Introduction 建立问题和缺口，不能提前假定研究结论。
- Methods 说明做了什么和为何这样做，不能用结果合理化方法。
- Results 报告预定和实际分析，不用 Discussion 语言解释。
- Discussion 先回答研究问题，再比较证据、解释限制和外推边界。
- 报告规范是最低透明度检查，不是段落模板。
- 用户修改要求优先保留原判断、证据和限制。
- 没有作者语料时使用克制的学术注册，不冒充个人指纹。
- 引用、作者贡献、利益冲突和 AI 使用按目标期刊当前规则核验。
