# RW PhD Write 方法

## 输入门

- 保留用户原问题和原始材料。
- 确认交付物、研究类型、可用证据和不能越过的范围。
- 缺少会改变判断的信息时，只追问必要项；其余内容继续处理。

## 执行阶段

1. 确认章节类型、章节功能、读者、目标主张和允许使用的材料。
2. 建立 claim-to-source 表，区分事实、作者解释、当前推断和未知事项。
3. 按章节功能组织论证移动、段落顺序和限制位置。
4. 先写论点和证据骨架，再写正文，不生成未核验引用。
5. 按研究类型检查相应报告规范、方法透明度和披露事项。
6. 运行来源、因果强度、前后一致性、作者语气和修改范围检查。

## 证据门

- 每项事实能回到用户材料或可核验公开来源。
- 推断标记为推断，缺失标记为缺失，不用常识补成已知事实。
- 当前信息在使用当日核验；稳定方法使用 Skill 内的摘要。
- 报告规范、偏倚工具和研究设计各自承担不同功能。

## 输出契约

- 章节、段落或论证骨架。
- claim-to-source 表和引用缺口。
- 修改说明、未解决问题和合规检查。

输出同时标记：已确认事实、当前判断、来源、未知事项、停止条件和下一步。

## 停止条件

- 没有来源时，不生成事实性引用或伪造 DOI。
- 没有研究结果时，不写成研究已完成。
- 不为流畅删除限制、反例和不确定性。
- 不把 Skill 输出当成作者最终责任或期刊合规确认。
