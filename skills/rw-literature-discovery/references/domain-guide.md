# RW Literature Discovery 选择表

| 情形 | 执行动作 | 不要做 |
|---|---|---|
| 生物医学问题 | 优先 PubMed，再用 Crossref 或 OpenAlex 补元数据和引用链 | 只搜网页 |
| 跨学科主题 | 使用至少两个覆盖不同的数据库 | 把单库零结果当结论 |
| 已知一篇种子论文 | 做前向、后向和相似文献追踪 | 只改关键词 |
| 需要 DOI 核验 | 用 Crossref 和出版商页面交叉核对 | 根据标题猜 DOI |
| 任务属于系统综述 | 转入 rw-review-methods 固化协议 | 把发现性检索冒充系统检索 |

## 核心规则

- 文献发现服务于判断，不以文献数量作为完成标准。
- 探索性检索和可复现系统检索要分开记录。
- 单一数据库的缺失不能解释为没有研究。
- 题名相似不能替代 DOI、PMID 或出版元数据核验。
- 摘要只能支持摘要中明确报告的内容。
- 预印本、会议摘要、期刊论文和撤稿记录要区分状态。
- 引用次数可以帮助发现入口，不能直接证明研究质量。
- 搜索式要保存原样、日期、平台和返回数量。
- 同义词扩展要与结果噪声一起测试。
- 遇到反常结果时要搜索替代解释和否定证据。
- API 失败、限流或覆盖不足时要有人工检索回退。
- 正式文献清单中的每项都要说明它支持、限制或改变哪个判断。
